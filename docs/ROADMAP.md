## v0.4c — parser-free CLI wiring (COMPLETE 2026-07-16)

Goal: make the already-shipped parser-free copy analysis usable from the public audit CLI without changing the default audit path or widening the copy-analysis claims.

1. Add explicit `--copy <copy-style.yaml>` parsing and help text. No flag means no config discovery, no copy findings/notices, and no `@design-harness/copy-audit` entry in metadata. Reject unknown or duplicate CLI options so a misspelled copy flag cannot silently run a no-copy audit.
2. In `@design-harness/cli`, preflight one explicit local path as a regular file, use a non-blocking open flag where the runtime exposes it, require the opened handle to still identify a regular file, read at most 1 MiB + 1 byte, reject input over 1 MiB and malformed/ambiguous UTF-8 YAML, validate the parsed value with the existing core `copy-style` schema, and only then pass the typed value to `auditUrl({ copyStyle })`. Read/parse/schema errors exit `1` before browser launch or audit artifacts.
3. Keep the existing programmatic integration as the single analysis path: copy findings remain pre-score, notices remain non-failing, output integrity/report rendering stay unchanged, and metadata records `@design-harness/copy-audit` only when configured.
4. Add args/loader/orchestration tests, extend packed-CLI smoke for the flag and fail-closed config errors, and add a live CLI fixture run while retaining the current programmatic copy smoke and calibration gates.
5. Update usage docs and a schema-valid example config without changing public positioning claims.

Explicitly out of scope: Kiwi or any morphological parser; register/josa-batchim/lemma expansion; spelling providers or dictionaries; hosted services or judges; `--locale` inference/mismatch behavior; copy-style auto-discovery; schema/enums/output-contract changes; new findings, criteria, or scoring rules; version bump, publish, tag, or GitHub Release.

Dependency gate resolution: on 2026-07-16 the owner approved `pnpm --filter @design-harness/cli add yaml@^2.9.0`; `yaml` 2.9.0 is a direct CLI-only dependency. `check:package-boundaries` rejects YAML dependencies/imports in core, copy-audit, and visual-audit; YAML parsing and user-supplied config file I/O remain CLI-only.

Acceptance: no-flag behavior is regression-locked; valid YAML drives the existing five parser-free checks and copy tool-version provenance; invalid/missing/ambiguous/schema-invalid config fails before audit side effects; args tests, loader/orchestration tests, live copy smoke, calibration, packed-CLI smoke, and `CI=true pnpm release:check` pass. Detailed execution plan: `.omx/plans/RALPLAN_V04C_PARSER_FREE_CLI_WIRING.md` (local planning artifact; may be absent from fresh clones).

## v0.5a — guide compile/check (COMPLETE 2026-07-18)

Goal: compile one explicit, schema-validated project guide into deterministic agent guidance and design tokens, then provide a non-mutating CI gate for drift and budget. The commands are `design-harness guide compile --guide <design-guide.yaml> [--copy <copy-style.yaml>] --target <project-dir>` and `design-harness guide check --guide <design-guide.yaml> [--copy <copy-style.yaml>] --target <project-dir> [--max-tokens <1..2000>]`; there is no discovery or network path.

1. **Supported generation profile `v0.5a-1`** — top-level generation fields are `schemaVersion: "0.2"`, `tokens`, `prohibitions`, and `signatureElement`. The DTCG-shaped token subset is 4–6 literal `srgb` semantic colors, exactly `heading`/`body` font families, and 2–12 literal `px`/`rem` values in each spacing/radius scale. Names are ASCII lower-kebab. Aliases/references, `$extends`, `$root`, composites, gradients, imports, themes, token metadata, and arbitrary input `$extensions` are rejected explicitly. `prohibitions` selects 1–8 IDs from the versioned `datasets/slop-fingerprints.json` guidance catalog; this catalog is not a detector or a universal-quality score. v0.5b adds one optional audit-only subtree without changing this generation projection. See `examples/configs/design-guide.example.yaml`.
2. **Deterministic projections** — the same normalized source produces a fixed name/description/bad→good pack in `AGENTS.md` and `DESIGN.md`, a non-duplicating `CLAUDE.md` `@AGENTS.md` shim, and sorted `design.tokens.json` with the `dev.design-harness` ownership extension. AGENTS/DESIGN use exactly `<!-- design-harness:guide:begin -->` and `<!-- design-harness:guide:end -->`; a standalone Claude import remains byte-identical. User content outside owned markers is preserved.
3. **Fail-closed materialization** — target and config paths must resolve inside the real, already-existing `--target`, without symlink traversal; an outside-target config fails at phase `containment` with an actionable target-relative diagnostic. Before any final mutation, compile validates every input/output, sanitizer, contradiction, budget, marker, and ownership condition. A canonical private lock serializes cooperating guide compiles; verified stages live inside it, existing destinations move to private recovery, and same-device hard links provide conditional no-overwrite commit/restore. Config reads are bound to their contained identities, and target/config/stage/output identities are checked around commit operations. The primary error remains primary; rollback/cleanup failures and preserved residue are secondary evidence. The handled-error guarantee covers concurrent changes observable across those guards; it does not claim global atomicity, crash recovery, or resistance to a hostile parent-directory ABA completed entirely within one path-based syscall window.
4. **Budget and check contract** — `guide-token-estimate-v1 = max(Unicode scalar count, ceil(UTF-8 bytes / 2))`; it is an estimate, never an exact model-token count. Compile enforces the hard ceiling 2000. Check performs zero writes, verifies all owned artifacts are current, and may only lower that ceiling with `--max-tokens 1..2000`. Invalid guide commands exit `1`; audit's partial exit `2` remains audit-only.
5. **Compatibility claim gate** — no published package gains a Style Dictionary or tokenizer runtime dependency. The repository tests the v0.5a profile with exact `style-dictionary@5.5.0` in a bounded CSS smoke; Style Dictionary remains a root development dependency only. This measured compatibility does not establish full DTCG 2025.10 conformance.

Acceptance: profile/sanitizer/contradiction/budget tests; catalog source/generated parity; marker, ownership, containment, race, staged-write, rollback, and cleanup-primary-error tests; first-run/idempotent/drift/no-write temporary-project smoke; audit/no-copy regressions; packed-CLI smoke; `CI=true pnpm validate`; and `CI=true pnpm release:check`. No version, publish, tag, release, root positioning, audit schema, criterion/finding/enum, capture, report, or scoring change belongs to v0.5a.

## v0.5b — font-family adherence + dogfood repair (COMPLETE 2026-07-20)

Goal: reuse one explicit, strictly validated `design-guide.yaml` at audit time and report only rendered text whose computed `font-family` list contains a member outside the guide's declared heading/body plus audit-only union. This proves conformance to a declared computed list, not the face used to render every glyph.

1. **One explicit policy path** — `design-harness audit --url <local-url> --guide <design-guide.yaml>` performs no discovery. The CLI validates the guide before browser or output work, projects only allowed family values, selector exceptions, and policy ID, and may coexist with explicit `--copy`. No guide means no font-policy loading, capture, findings, notices, metadata, or failed checks.
2. **Audit-only guide overlay** — `audit.fontFamily` is a closed non-empty object with independently optional `additionalAllowedFamilies` and `ignoreSelectors`. The additional list contains 1–32 exact decoded `{ value, kind: named|generic }` members of 1–128 safe Unicode scalars; generic entries must name a supported CSS generic, while named entries may deliberately use a generic-looking spelling. Selectors retain their existing 1–32, 256-scalar bounds and exclude matching third-party subtrees from this check only. The v0.5a generation projection remains exactly the original four fields; all overlay-only changes are byte/hash/provenance inert.
3. **Frozen comparison contract** — `font-family-adherence-v1` unions heading, body, then additional values without role or stack-order enforcement, deduplicating by kind plus ASCII-folded value while preserving first spelling. Named families and CSS generics remain distinct; every non-ASCII code point compares exactly without Unicode normalization, locale rules, or `Intl.Collator`. No Next, suffix, hash, alias, prefix, glob, regex, or first-member rule receives automatic trust.
4. **One bounded computed-style risk** — `unapproved-font-family` maps to `visual.font-family.project-contract`: deterministic `project-contract` risk, `computed-style`, high confidence, low severity. Only existing visible text candidates are evaluated. Violations group by raw computed stack per viewport and emit at most five findings; diagnostics record evaluated/ignored/violating and distinct/emitted/truncated stack counts without claiming actual glyph-face usage.
5. **Scoped partial semantics** — selector-engine or computed-family processing errors affect only this check. Base measurements and unrelated findings survive; font evidence from the failed scope is discarded, the audit records a stable failed check, and normal partial-exit behavior applies.

Repair acceptance: strict additional-only/selector-only/combined schema and projection tests; frozen generation hashes and guide drift; criterion-policy and integrity gates; unchanged CLI/visual production seams; live long-stack good plus declared `Rogue`/undeclared `Rogue Fallback` negative fixtures; packed additional-only and fail-before-output config paths; pinned Observed `/providers/openai` at `991ad20644b1d7d28f3833b1017ca0b9af115848` and 정책한눈 `/` at `d08d89cc8a3f89ecb13fc1d7337551a555ed85ac`, each with zero selectors/violations; `CI=true pnpm validate`; and `CI=true pnpm release:check`. No dependency, enum, schema-version, policy-ID, scoring, report-contract, version, publish, tag, or release change belongs to this repair.

## Surface horizons (web → app → beyond) — NOT milestones

Principles and enforcement: ADR-002 (`docs/adr/ADR-002-surface-extensibility-principles.md`, `check:core-purity`). Evidence and per-item citations: `docs/research/2026-07-surface-extensibility-evidence.md`. Nothing here is buildable without its gates passing AND the owner scheduling it into a milestone; this section exists so architecture decisions never foreclose these paths. Expansion gates (all required per surface): a real file-contract consumer asks for it · v0.5 reins loop proven on web · evidence-layer contract shipped · that surface's open questions resolved by hands-on probe.

- **Tier 0 — reachable today, zero capture code** (demonstrations, not new surfaces): Storybook story iframes (`/iframe.html?id=…`; needs a component-mode check profile — page-scoped checks misfire on bare components); design-system docs/marketing pages; PWA as plain web; React Native Web / Expo web builds (audits the web projection, never claims about the native binary); in-page data-viz (input side; any Chartability-derived check family is heuristic/needs-review tier, CC-BY-SA 3.0 diligence).
- **Tier 1 — small adapter, localhost preserved**: Playwright device-emulation profile (responsive layout math only; report copy must never imply real-device or iOS-Safari-engine claims); Electron/WebView2 via local CDP endpoint (Chromium DOM → near-100% check transfer; widening the input contract to ws://localhost needs its own ADR).
- **Tier 2 — native mobile apps**: a documented evidence-bundle **ingest format** (semantic tree + screenshot + capture-integrity metadata) that external Appium/adb/XCUITest/Maestro scripts produce — we never own a device loop. Android before iOS (Roborazzi/ATF runs emulator-free on the JVM; iOS always needs a simulator). Standards anchors: WCAG 2.2 via WCAG2ICT (Group Note 2024-10) / WCAG2Mobile (still Draft Note — verify before building), EN 301 549 Annex C (normative tests; v4.1.1 OJEU ~Oct 2026 unconfirmed), KS X 3253 + 디지털포용법 (2026-01-22) for Korea — certification is part human-judged, so KS-derived checks need per-item triage. Rendered text inventory survives this bridge → Korean copy checks stay valid; computed-style checks do not.
- **Tier 3 — different artifact classes, each needs its own input-policy ADR**: PDF via veraPDF delegation (PDF/UA = the second official-testable deterministic-failure family; file input breaks hard rule 8; Java dep); email templates — honest subset only: rendered-copy checks (Korean josa post-interpolation) + deterministic template semantics (`lang`/`dir`/`role=presentation` — EMC 2026: 99.88% of 376k emails fail, top-3 failures are these), claims scoped to template source, client-compat as versioned caniemail data at heuristic tier; never client-rendering claims (Outlook = Word engine), never screenshot farms. Korean fit is strong (Stibee ecosystem has no QA layer) but neither current goal centers email — owner decision required.
- **Evidence-against (do not build)**: owned mobile capture / device farm · Windows UIA / macOS AXAPI desktop-native surface · Tauri-on-macOS (no WKWebView WebDriver) · slide decks (no open deterministic checker) · PWA-specific checks (Lighthouse 12 removed the category) · email client screenshot farm · Figma REST importer before the plugin-surface evidence-against is re-examined (networked input, no semantics in the artifact).

- Best-of-N candidate picker; community fixture pipeline (users follow demos, contributors follow users); more than two agent surfaces (Claude Code + Codex).

- `guide from-references` VLM-distiller CLI before the manual workflow proves value.
- Open Design integration (spec-only).
- **Evidence-against (peer-reviewed) — do not build**: hue-template color-harmony scoring (O'Donovan: no predictive power; 52% inter-rater agreement — use lightness-contrast structure instead); mirror-symmetry/balance scoring for real UIs (validates only on abstract stimuli); scored Korean readability (KRIT 0.746 on long-form textbook text; nothing validated for short strings — raw informational metrics at most); MQM translation-accuracy LQA product (owned by Lokalise/Phrase/Crowdin); Figma-plugin surface (crowded); generic English style-guide enforcement (Writer/Markup AI); Bareun or any hosted API in default paths (lapse test).
